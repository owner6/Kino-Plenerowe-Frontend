# Kino Plenerowe — Frontend

Веб-додаток для перегляду репертуару кіно під відкритим небом. Користувачі можуть обирати дату, переглядати сеанси на обраний день, переходити на сторінки локацій та переглядати їх на карті.

Проєкт працює разом із **backend API** (REST), який надає події, локації та дані для Google Maps.

## Основні можливості

### Репертуар за датою

- Горизонтальний тижневий календар (7 днів від сьогодні)
- Список кінопоказів на обрану дату: фільм, час, місце, адреса, ціна
- Перехід на сторінку локації з картки події

### Синхронізація з URL

- Сьогоднішня дата: `/`
- Конкретна дата: `/2024-06-15` (формат `YYYY-MM-DD`)
- Підтримка кнопок «назад/вперед» у браузері
- Динамічні SEO-метадані (`title`, `description`) для сторінок з датою

### Локації

- Список усіх місць проведення: `/places`
- Сторінка локації: `/places/:slug` — опис, майбутній репертуар, адреса
- Вбудована Google Maps (iframe з backend)
- Локальні текстові описи локацій (fallback, якщо API не повертає опис)
- SEO-дані локації з API (`seo_title`, `seo_description`)

### Навігація

- Верхнє меню: головна, локації
- У меню також є посилання на розклад, карту та «про нас» — маршрути ще не реалізовані

| Маршрутизація | Vue Router 4                              |

| Збірка        | Vite 7                                    |
| Node.js       | `^20.19.0` або `>=22.12.0`                |

## Структура проєкту

- **`src/views/`** — сторінки додатку
  - `HomeView.vue` — головна (календар + події на дату)
  - `PlaceView.vue` — список локацій і сторінка місця
- **`src/components/`** — UI-компоненти
  - `common/` — календар, карта
  - `movies/` — список подій
  - `layout/` — навігація
- **`src/services/`** — робота з API та локальні дані
  - `eventsService.js` — запити до backend
  - `placeDescriptionsService.js` — описи локацій
- **`src/router/index.js`** — маршрутизація
- **`src/main.js`** — ініціалізація Vue-додатку

## Backend API

Frontend очікує REST API за адресою `http://localhost:3000` (налаштовується в `src/services/eventsService.js`).

| Метод | Endpoint                          | Опис                                  |
|-------|-----------------------------------|---------------------------------------|
| GET   | `/events`                         | Усі події                             |
| GET   | `/places`                         | Список локацій                        |
| GET   | `/places/:slug?include=seo`       | Деталі локації                        |
| GET   | `/places/:slug/events`            | Події локації (`upcoming`, `past`)    |
| GET   | `/places/:slug/map-iframe-url`    | URL iframe Google Maps                |

> Перед запуском frontend переконайтеся, що backend працює на порту 3000 (або оновіть `API_BASE_URL`).

## Встановлення та запуск

```bash
# Встановлення залежностей
npm install

# Режим розробки (http://localhost:5173)
npm run dev

# Збірка для продакшену
npm run build

# Попередній перегляд production-збірки
npm run preview
```

## Скрипти

| Команда           | Опис                          |

## Маршрути

| Шлях              | Компонент  | Опис                              |
|-------------------|------------|-----------------------------------|
| `/`               | HomeView   | Сьогоднішній репертуар            |
| `/:date`          | HomeView   | Репертуар на дату (`YYYY-MM-DD`)  |
| `/places`         | PlaceView  | Усі локації                       |
| `/places/:slug`   | PlaceView  | Деталі локації                    |
